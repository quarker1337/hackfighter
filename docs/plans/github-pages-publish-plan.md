# GitHub Pages Publishing Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Publish the Godot Web export of HACKFIGHTER 2 to GitHub Pages so the game is playable at `https://quarker1337.github.io/hackfighter/` after merges to the main branch.

**Architecture:** Use GitHub Actions to build the Godot Web export in CI, upload `export/web` as a static Pages artifact, and deploy through GitHub Pages. Keep generated Web build files out of the source branch; the source branch remains clean and `export/` stays ignored. GitHub Pages provides HTTPS, which satisfies Godot Web secure-context requirements.

**Tech Stack:** Godot 4.6.2, Godot Web export templates, GitHub Actions, GitHub Pages, static HTML/WASM/PCK hosting.

---

## Is this possible?

Yes, sir. This is possible and is a good fit for GitHub Pages because the Godot Web export is a static site:

- `index.html`
- `index.js`
- `index.wasm`
- `index.pck`
- audio worklet files
- `audio_bridge/` files generated by `tools/export_web.sh`

GitHub Pages serves static files over HTTPS, so the browser secure-context requirement is satisfied. We should keep COOP/COEP disabled because this project does not need SharedArrayBuffer/thread isolation and COEP can break subresource loading in iframes or static hosts.

Expected public URL after setup:

```text
https://quarker1337.github.io/hackfighter/
```

---

## Key decisions

1. Use GitHub Actions deployment, not a committed `export/` folder.
   - Reason: `export/` is generated build output and already ignored.
   - Result: clean source history; GitHub builds the playable static site.

2. Patch `tools/export_web.sh` so CI can select the Godot binary.
   - Current script hardcodes `~/bin/godot4`.
   - In GitHub Actions we need something like `GODOT_BIN="$PWD/.ci/godot/Godot_v4.6.2-stable_linux.x86_64" ./tools/export_web.sh`.

3. Add `.nojekyll` to the generated `export/web` folder.
   - Reason: Pages should serve files exactly as exported and not run Jekyll processing.

4. Verify the deployed game visually before declaring victory.
   - For this project, proof means the public URL loads the audio gate/menu, then the HACKFIGHTER menu appears after click/Enter.

## Task 3: Add GitHub Pages deployment workflow

**Objective:** Build and deploy the Web export automatically from GitHub Actions.

**Files:**
- Create: `.github/workflows/deploy-pages.yml`

**Workflow contents:**

```yaml
name: Deploy Godot Web to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Configure Pages
        uses: actions/configure-pages@v5

      - name: Cache Godot export templates
        uses: actions/cache@v4
        with:
          path: ~/.local/share/godot/export_templates/4.6.2.stable
          key: godot-4.6.2-web-export-templates-v1

      - name: Install Godot 4.6.2 and Web export templates
        shell: bash
        run: |
          set -euo pipefail
          mkdir -p .ci/godot
          cd .ci/godot
          curl -L -o godot.zip "https://github.com/godotengine/godot/releases/download/4.6.2-stable/Godot_v4.6.2-stable_linux.x86_64.zip"
          unzip -o godot.zip
          chmod +x Godot_v4.6.2-stable_linux.x86_64

          TEMPLATE_DIR="$HOME/.local/share/godot/export_templates/4.6.2.stable"
          if [ ! -f "$TEMPLATE_DIR/web_release.zip" ]; then
            mkdir -p "$TEMPLATE_DIR"
            curl -L -o godot_web_templates.tpz "https://github.com/godotengine/godot/releases/download/4.6.2-stable/Godot_v4.6.2-stable_export_templates.tpz"
            python3 - <<'PY'
import zipfile
from pathlib import Path
out = Path.home() / ".local/share/godot/export_templates/4.6.2.stable"
with zipfile.ZipFile("godot_web_templates.tpz", "r") as z:
    z.extractall(out)
PY
            if [ -d "$TEMPLATE_DIR/templates" ]; then
              mv "$TEMPLATE_DIR"/templates/* "$TEMPLATE_DIR"/
              rmdir "$TEMPLATE_DIR/templates"
            fi
          fi

      - name: Build Godot Web export
        shell: bash
        run: |
          set -euo pipefail
          GODOT_BIN="$PWD/.ci/godot/Godot_v4.6.2-stable_linux.x86_64" ./tools/export_web.sh
          test -f export/web/index.html
          test -f export/web/index.wasm
          test -f export/web/index.pck
          test -f export/web/.nojekyll
          grep -q '"ensureCrossOriginIsolationHeaders":false' export/web/index.html

      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: export/web

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

**Verification before commit:**

Run locally:

```bash
git diff --check
```

Expected: no output.

**Commit:**

```bash
git add .github/workflows/deploy-pages.yml
git commit -m "ci: deploy godot web build to github pages"
```

## Risks / gotchas

1. GitHub Pages cannot serve custom COOP/COEP headers.
   - This is okay for us because the project should not require threads/SharedArrayBuffer.
   - Keep `ensureCrossOriginIsolationHeaders=false`.

2. Godot version/template mismatch can break the Web build.
   - Use Godot 4.6.2 binary and 4.6.2 export templates together.

3. `tools/export_web.sh` currently assumes `~/bin/godot4`.
   - Fix with `GODOT_BIN` before adding CI.

4. GitHub Actions downloads are large.
   - Cache export templates to avoid repeatedly downloading the full templates archive.

5. GitHub Pages URL for project repos uses a subpath.
   - Expected URL is `/hackfighter/`, not the domain root.
   - Godot export uses relative asset paths, so this should work.

6. Audio must start from a real user gesture.
   - Our Web export already has a click/Enter audio gate; keep it.

7. The first workflow run may fail if Pages is not enabled for Actions.
   - Enable `Settings -> Pages -> Source: GitHub Actions`.
